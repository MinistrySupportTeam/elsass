    elsass-gutters

    $elsass-gutters: (
  'above'   : ('out': ('top': 1)), // Uses the full gutter length as the top margin.
  'after'   : ('out': ('right': 1)), // …
  'below'   : ('out': ('bottom': 1)),
  'before'  : ('out': ('left': 1)),
  'outside' : ('out': 'none' .5), // Uses a half gutter length as right and left margins.
  'all-out' : ('out': 'none' .5 1 .5), // Adds a full gutter length as the bottom margin.
  'up'      : ('in' : ('top': 1)), // Uses the full gutter length as the top padding.
  'end'     : ('in' : ('right': 1)), // …
  'down'    : ('in' : ('bottom': 1)),
  'start'   : ('in' : ('left': 1)),
  'inside'  : ('in' : 'none' .5),
  'all-in'  : ('in' : .5),
  'all-over': ('in' : .5, 'out': 'none' .5 1 .5) // Uses paddings and margins.
) !default;
    View source

    Description

    Custom gutters related keywords and their values.
    Allows to easily set a gutter value through elsass-global or the elsass mixin $gutter argument by providing a key.
    none and default keys are not allowed.

    Type

    Map

    Map structure

    Map key NameMap key DescriptionMap key TypeMap key Value
    gutter

    Custom gutter related map of 'in' (padding) and/or 'out' (margin) gutter(s).
    Each key set accepts one of the following values:

    • a map of side(s) related gutter length ratios or <length>;
    • a list of up to four gutter length ratios or <length> sorted in the same way as the padding or margin shorthand value and where any unwanted side(s) related gutter can be set to none;
    		Map— none

    elsass-behaviours

    $elsass-behaviours: (
  'silent'  : (
    'in'    : ('in' : 'none'), // Used when the current gutter is set via the padding.
    'out'   : ('silent'), // Used when the current gutter is set via the margin.
    'in-out': ('silent') // Used when the current gutter is set via padding and margin.
  ),
  'skip'    : (
    'in'    : ('in' : 'none'), // …
    'out'   : ('out': 'none'),
    'in-out': ('in' : 'none', 'out': 'none')

  'collapse': (
    'in'    : ('in' : 0),
    'out'   : ('out': 0),
    'in-out': ('in' : 0, 'out': 0)
  ),
  'nest'    : (
    'in'    : ('out': 'in' -1),
    'in-out': ('out': 'in' -1)
  ),
  'bleed'   : (
    'in'    : ('in': 'in' 2, 'out': 'in' -1),
    'out'   : ('in': 'out' 1, 'out': 'out' -1),
    'in-out': ('in': 'none', 'out': 'out' -1)
  ),
  'fit'     : (
    'in'    : ('in': 'none'),
    'out'   : ('out': 'out' -1),

    Description

    Calculates a span width.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $width

    Defines the length base.

    • <number> defines a default columns number fraction;
    • <number> of <number> defines a custom columns number fraction;
    • max uses the global container max-width set.
    • <breakpoint> uses the breakpoint related container max-width set.
    		Number or String— none
    $gutter-width

    Defines the margin length to substract from one of the following value.

    • a sum or list of the right and left gutter ratios or lengths;
    • a map using a breakpoint as key and a sum or list of the right and left gutter ratios or lengths as value.
    • a breakpoint name (uses the related gutter lentgh);
    		Number or List or Map— none

    Returns

    Number —

    <length> (via calc() if needed).

    Example

    SCSS

    width: elsass-width(12, 'm');
max-width: elsass-width('m', 'm');

    elsass

    @mixin elsass($media: from $ELSASS-BPT-MIN, $width, $gutter, $offset, $extra) { ... }
    View source

    Description

    Builds any responsive element.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $media

    Defines the related media-query/ies min and\or max-width from one of the following value:

    • <breakpoint> applies to the provided breakpoint only;
    • from <breakpoint> applies to the provided breakpoint and larger ones;
    • to <breakpoint> applies to smaller breakpoints than the one provided;

    • <breakpoint> to <breakpoint>.

      Each value using from or to allows <breakpoint> to be replaced by a <lenght> only if no breakpoint settings related value is needed.

    		Listfrom $ELSASS-BPT-MIN
    $width

    Defines the element width and (optionally) max-witdh from one of the following values.

    • <number> defines a default columns number related width or a max-width related to the global/breakpoint(s) container max-width set if followed by max;
    • <number> of <number> defines a custom columns number related width or a max-width related to the global/breakpoint(s) container max-width set if followed by max;
    • max set a max-width equal to the global/breakpoint(s) container max-width set.
    		List— none
    $gutter

    Applies gutters as padding and/or margin based on the following valid values.

    • A map, as defined in $elsass-gutters;

    • A shorthand as

      • default: uses the default gutter set;
      • none: does not apply any gutter;
      • an $elsass-gutters key: applies the related value;
      • an $elsass-behaviours key: applies the related change(s) to alterate the default gutter;
      • an $elsass-gutters key followed by an $elsass-behaviours key: applies the $elsass-behaviours related change(s) to alterate the $elsass-gutters related value.

      $elsass-behaviours keys accept suffixes to alterate defined gutter sides. Valid sufixes are -top, -right, -bottom, -left, -x and -y.
      If no suffix is provided all gutter related sides will be alterate.

    		List or Map— none
    $offset

    Alterates the horizontal element and/or content position via margin and/or padding through the following values.

    • A map of in and/or out offset values as:
      • a map of sides related or grid fraction or <length>;
    • A shorthand using a list of $elsass-offsets keys optionally followed by a related grid fraction or <length>. If no ratio is provided auto will be used as the padding/margin value.
    		List or Map— none
    $extra

    List of extra CSS rules to apply.
    Gutter or width related CSS values can be indirectly set by providing a map using gutter or width as key and a list or map of the elsass-gutter or elsass-width function arguments as value.

    		List or Map— none

    Example

    SCSS

    .foo {
  border-style: solid;

  @include elsass('m', 6, 'after', 'before' 3, (border-width: ('gutter': 1)));
}

    Output

    Arguments and @content relative CSS wrapped into tidy media-queries.

    elsass-group

    @mixin elsass-group($config) { ... }
    View source

    Description

    Groups multiple elsass mixins arguments lists/maps to optimize the output by avoiding duplicated CSS through media-queries.

    Parameters

    parameter Nameparameter Descriptionparameter Typeparameter Default value
    $config

    elsass mixin arguments lists/maps (see elsass).

    		List— none

    Example

    SCSS

    .foo {
  border-style: solid;

  @include elsass-group(
    ('extra': (border-width: ('gutter': .5))) // Uses a responsive gutter value as the border-width value.
    ('s', 12, 'default') // A twelve columns large span for the 's' breakpoint.
    ('m' to 'xl', 6, 'default') // A six columns large span for breakpoints 'm' and 'l'.
    (from 'xl', 4, 'default') // A four columns large span for greater breakpoints.
  );
}
